Data Structures 101

큐(Queue) vs 스택(Stack)

왜 우리 시스템은 '큐' 방식을 사용할까요?

Queue
System Choice

일상 속 비유

은행 번호표 / 줄서기

FIFO (First-In First-Out)

1
2
3
4

먼저 온 작업이 먼저 처리됨

  • 선입선출 (FIFO):
    순서가 보장되어 예측 가능합니다.

  • 공정성:
    먼저 요청한 사용자가 먼저 결과를 받습니다.

Stack
참고용 (Reference)

일상 속 비유

접시 쌓기 / 엘리베이터

LIFO (Last-In First-Out)

Data 1
Data 2
Data 3

마지막에 넣은 것이 먼저 나옴

  • 후입선출 (LIFO):
    나중에 들어온 요청이 새치기하는 셈입니다.

  • 부적합:
    먼저 요청한 작업이 무기한 대기할 수 있습니다.

결론: 이미지 처리 순서를 보장하기 위해 '큐(Queue)'를 사용합니다.
